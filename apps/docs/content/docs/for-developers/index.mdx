在进行 Valhalla 的拓展开发之前, 我们需要了解 Valhalla 的架构和设计理念。

## 目录结构

我们使用 Turborepo 来管理我们的项目, 以下是我们的目录结构, 我们已为您忽略了一些文件和文件夹, 这些文件和文件夹通常无需关注。

现在, 让我们来逐一探索这些文件和文件夹。

### `apps/docs`

这是 Valhalla 的文档站点, 使用 Next.js 构建。

### `apps/web`

这是 Valhalla 的核心应用, 使用 Next.js 构建。

### `examples/common`

这是 Valhalla 的示例资源, 如果你需要快速开发一个 Valhalla 资源而不打算发布, 可以参考这个示例。

### `packages/components`

这是 Valhalla 的组件库, 基于 Radix UI。

### `packages/database`

这是 Valhalla 的数据库管理工具, 基于 Prisma。

### `packages/resource`

这是 Valhalla 的资源类型声明以及资源创建工具。

### `packages/resource-components`

这是 Valhalla 的资源组件库, 基于 Radix UI, `packages/resource` 以及 `packages/components`。

### `.env`

这是 Valhalla 的环境变量文件, 用于配置 Valhalla 的运行环境。

### `package.json`

这是 Valhalla 的包管理文件, 用于配置 Valhalla 的依赖和脚本。

### `start-database.sh`

这是 Valhalla 用于开发时启动数据库的脚本, 你需要先安装 [Docker](https://www.docker.com/) 并启动 Docker 服务。
我们不建议在生产环境中使用此脚本, 生产环境中请使用 [Docker Compose](https://docs.docker.com/compose/) 来启动数据库。
